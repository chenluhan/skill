# Page Spec Template

Use this fixed structure for every page in `02-pages.md`.

## Section Order

```markdown
## <Page Name>

### Page Goal
- ...

### Evidence
- 明确事实:
  - ...
- 稳定推断:
  - ...

### Entry Points
- ...

### Exit Points
- ...

### Information Blocks
- ...

### User Actions
- ...

### System Feedback
- ...

### States
- ...

### Exceptions
- ...

### Permissions
- ...

### Analytics
- ...

### Low-Fidelity Wireframe
```text
+----------------------------------+
| Header                           |
+----------------------------------+
| Body section                     |
| Primary CTA                      |
+----------------------------------+
```
```

## Rules

- Do not skip the `States` section.
- Do not skip `Exceptions` just because the source only shows the happy path.
- Put uncertainty in `Evidence` or `05-open-questions.md`, not inside the wireframe.
- Keep wireframes structural. Do not describe pixel values or visual styling.
