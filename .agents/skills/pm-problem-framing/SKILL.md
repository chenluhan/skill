# PM Problem Framing

Goal: move from “something feels important” to “this is the exact product problem to solve now.”

## Require

- A `Normalized Brief`
- Current product context
- Existing user behavior or workflow
- Any known business constraints

## Produce

- Target users
- Core scenario
- Current job and workaround
- Main obstacle
- Why now
- Success metrics
- Non-goals

## Run This Workflow

1. Name the target user specifically. Avoid “all users”.
2. Describe the scenario where the problem appears.
3. Reconstruct how the user currently completes the job.
4. Explain why the current path is insufficient.
5. State why now matters.
6. Define measurable success.
7. Write explicit non-goals before any prototype work begins.

## Hold These Gates

- Do not let screen ideas replace problem definition.
- Do not use vague success statements such as “improve experience”.
- Do not continue if non-goals are missing.

## Recover From Failure

- If user and scenario stay fuzzy, go back and gather more context.
- If metrics are not available, choose a proxy metric and label it.
- If scope debates dominate this step, return to the problem and simplify it.

## Hand Off

Pass the result to `$pm-scope-cut`.
