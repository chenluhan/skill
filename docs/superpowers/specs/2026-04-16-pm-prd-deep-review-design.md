# pm-prd-deep-review 设计说明

## 目标

创建一个面向 App 产品需求文档的深度诊断 skill。它的职责不是泛泛点评，而是系统识别 PRD 在以下方面的缺失与风险：

- 用户目标与主链路是否闭环
- 页面与交互是否足以支撑任务完成
- 状态矩阵是否完整
- 异常、边界、权限、弱网、中断恢复是否缺失
- 埋点、指标、业务规则是否能支撑验证与上线
- 当需求涉及 AI 时，是否补充了不确定性、失败兜底与成本约束

输出默认是一份可直接交给产品经理继续修订的 Markdown 审查报告。

## 适用范围

- 主场景：App PRD
- 常见类型：增长、会员、工具、内容、社区、交易、混合型需求
- AI 相关：可选扩展模块，不作为默认主检查维度

不适合的场景：

- 纯技术方案设计文档
- 仅一句话需求或非结构化聊天记录
- 需要直接产出最终 UI 方案而不是审查现有 PRD 的任务

## 核心定位

skill 名称：`pm-prd-deep-review`

核心定位：

1. 先重建需求的用户任务链路，再沿链路找断点
2. 再用横向检查补齐状态、规则、埋点、权限等常见漏项
3. 对 AI 相关能力追加专项检查，而不是对所有 PRD 强行套 AI 模板

## 触发信号

以下表达应触发该 skill：

- 帮我 review 这个 PRD
- 看看这份需求哪里有漏洞
- 检查交互、状态、异常有没有漏
- 审一下这份 App 产品方案
- 帮我做深度 PRD 诊断

## 工作流

### 1. 识别 PRD 类型

先判断当前 PRD 更接近哪类产品任务：

- 增长转化
- 内容消费
- 交易流程
- 工具效率
- 社区互动
- 会员与订阅
- 混合型

目标是避免用一套固定清单硬套不同产品。

### 2. 重建用户主任务链路

从文档中抽出：

- 目标用户
- 目标任务
- 触发入口
- 关键操作路径
- 成功结果
- 回流或留存动作

如果 PRD 无法还原出完整链路，应明确判定为结构不完整。

### 3. 纵向链路诊断

沿每个链路节点检查：

- 前置条件是否明确
- 用户能否理解当前状态
- 页面反馈是否足够
- 是否支持返回、撤销、重试
- 是否存在中断恢复
- 成功与失败后下一步是否清楚

### 4. 横向完整性补检

统一检查以下横向维度：

- 状态矩阵：空态、加载态、失败态、弱网态、权限拒绝态、重复操作态、中断恢复态
- 交互闭环：入口、跳转、返回、确认、取消、反馈
- 业务规则：边界值、频控、幂等、资格限制、角色差异
- 数据闭环：指标、埋点、口径定义、实验验证条件
- 文案引导：错误提示是否能指导下一步

### 5. AI 专项检查

仅在 PRD 明确涉及 AI 功能时启用，重点检查：

- 输入约束是否明确
- 输出是否允许不确定性
- 失败、超时、敏感内容是否有兜底
- 是否允许重试、编辑、人工接管
- 成本与延迟是否影响产品体验

### 6. 生成可执行修订建议

每条问题都需要输出：

- 问题描述
- 对用户或业务的影响
- 风险等级
- 建议补写到 PRD 的位置
- 参考补写方向

## 输出结构

最终 Markdown 报告固定包含以下部分：

1. `审查结论`
2. `关键致命问题`
3. `用户主链路诊断`
4. `状态与异常矩阵`
5. `交互与信息架构问题`
6. `业务规则与数据口径`
7. `AI 专项检查`（如适用）
8. `建议补写清单`
9. `可直接带进评审会的问题列表`

## 问题分级

每条问题必须打上以下标签之一：

- `致命缺失`
- `高优先级缺失`
- `中优先级优化`
- `信息待澄清`

## 资源设计

skill 至少包含以下文件：

- `SKILL.md`：核心工作流与触发说明
- `references/review-checklist.md`：深度审查清单
- `references/state-matrix.md`：状态与异常补检方法
- `references/output-template.md`：Markdown 输出模板

可选但推荐：

- `references/ai-review.md`：AI 功能专项检查

## 关键写作约束

- 不得臆测 PRD 未提供的事实
- 未说明的内容必须明确标注为缺失或待澄清
- 先给结论，再给证据
- 不只指出问题，还要说明用户影响和建议修法
- 优先识别会影响评审、交付、开发和上线验证的缺口

## 成功标准

该 skill 的成功，不是“写出很多意见”，而是：

1. 能快速指出 PRD 是否适合进入评审或开发
2. 能覆盖 App PRD 最常漏掉的关键状态与异常
3. 能输出足够明确的修订建议，让产品经理能直接补文档
4. 在不涉及 AI 的 PRD 上，也能保持高相关度和高可用性
